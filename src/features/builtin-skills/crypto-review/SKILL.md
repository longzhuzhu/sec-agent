# 密码学最佳实践审查

## 触发词
加密, 密码学, TLS, SSL, 哈希, 签名, 证书, crypto, encryption, hash

## 描述
审查代码中的密码学实现，确保符合当前安全最佳实践。

## 工作流程

### 步骤 1: 密码学使用审查

| 用途 | 推荐算法 | 禁止算法 |
|------|---------|---------|
| 对称加密 | AES-256-GCM, ChaCha20-Poly1305 | DES, 3DES, RC4, ECB模式 |
| 哈希 | SHA-256, SHA-3, BLAKE2 | MD5, SHA-1 |
| 密码哈希 | Argon2id, bcrypt, scrypt | MD5, SHA-*, 无盐哈希 |
| 非对称加密 | RSA-2048+, Ed25519, P-256 | RSA-1024, DSA |
| TLS | TLS 1.2+, TLS 1.3 | SSL 3.0, TLS 1.0/1.1 |
| 随机数 | crypto.randomBytes, CSPRNG | Math.random, rand() |

### 步骤 2: 密钥管理审查

- [ ] 密钥不硬编码在源码中
- [ ] 使用环境变量或密钥管理服务
- [ ] 密钥有合理的轮换机制
- [ ] 密钥长度满足安全要求
- [ ] 密钥派生使用 KDF（HKDF, PBKDF2）

### 步骤 3: 实现审查

- [ ] IV/Nonce 不重用
- [ ] 使用认证加密（AEAD）
- [ ] 恒时比较（防止时序攻击）
- [ ] 适当的填充方案
- [ ] 证书验证未被禁用

### 步骤 4: 生成审查报告

每个发现包含：
- 问题描述和 CWE 编号
- 当前实现代码
- 推荐的安全替代方案
- 迁移建议
