# 隐私合规检查

## 触发词
GDPR, CCPA, PIPL, 隐私, PII, 个人数据, 数据保护, privacy, compliance

## 描述
检查代码和系统设计是否符合主流隐私法规（GDPR/CCPA/PIPL）。

## 工作流程

### 步骤 1: PII 数据识别

扫描代码中的个人数据处理：

| 数据类型 | 示例 | 敏感等级 |
|---------|------|---------|
| 直接标识符 | 姓名、身份证号、邮箱 | 高 |
| 准标识符 | 年龄、地址、IP | 中 |
| 敏感数据 | 健康、财务、生物特征 | 极高 |
| 行为数据 | 浏览历史、位置轨迹 | 中 |

### 步骤 2: 数据流追踪

追踪 PII 从收集到销毁的完整生命周期：
1. **收集**: 用户输入、API接收、第三方同步
2. **处理**: 业务逻辑、分析、聚合
3. **存储**: 数据库、缓存、日志
4. **传输**: API调用、跨境传输
5. **销毁**: 删除机制、保留期限

### 步骤 3: 合规差距分析

#### GDPR 检查清单
- [ ] 合法处理基础（第6条）
- [ ] 数据最小化（第5条）
- [ ] 目的限制（第5条）
- [ ] 存储限制（第5条）
- [ ] 数据主体权利（第15-22条）
- [ ] 跨境传输（第44-49条）
- [ ] DPIA（第35条）
- [ ] DPO 任命（第37条）

#### CCPA 检查清单
- [ ] 隐私政策披露
- [ ] 消费者知情权
- [ ] 删除权实现
- [ ] 退出销售机制
- [ ] 未成年人保护

### 步骤 4: 生成合规报告

输出结构化合规报告，包含：
- PII 数据清单
- 数据流映射
- 合规差距列表
- 修复建议和优先级
