# API 安全测试方法

## 触发词
API安全, 认证, 授权, REST安全, GraphQL安全, api security

## 描述
系统化的 API 安全测试方法，覆盖认证、授权、输入验证等关键领域。

## 工作流程

### 步骤 1: API 枚举

1. 收集所有 API 端点（路由定义文件）
2. 标识认证要求（公开/私有/管理员）
3. 记录请求参数和响应格式
4. 识别文件上传端点

### 步骤 2: 认证测试

| 测试项 | 描述 |
|--------|------|
| 无认证访问 | 尝试不带 Token 访问受保护端点 |
| Token 篡改 | 修改 JWT payload 测试验签 |
| 算法混淆 | JWT alg=none 或 RS256→HS256 |
| Token 过期 | 使用过期 Token 测试 |
| 暴力破解 | 测试登录接口速率限制 |
| 会话固定 | 检查登录后是否更换 Session ID |

### 步骤 3: 授权测试

| 测试项 | 描述 |
|--------|------|
| IDOR | 修改资源 ID 访问他人数据 |
| 水平越权 | 同级用户间的数据隔离 |
| 垂直越权 | 普通用户访问管理接口 |
| 功能级 | 隐藏端点未做权限检查 |
| 批量赋值 | 提交额外字段（如 role=admin） |

### 步骤 4: 输入验证测试

- SQL/NoSQL 注入
- XSS（反射型/存储型）
- SSRF（内网探测）
- 路径遍历
- XML 外部实体（XXE）
- 参数污染

### 步骤 5: 速率限制和 DoS

- 测试请求速率限制
- 大 payload 处理
- 批量操作端点
- GraphQL 深度/复杂度限制

### 步骤 6: 生成测试报告

每个发现包含：
- 端点和方法
- 复现步骤
- 请求/响应示例
- 影响评估
- 修复建议
