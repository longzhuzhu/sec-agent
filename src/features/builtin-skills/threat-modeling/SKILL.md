# STRIDE 威胁建模方法论

## 触发词
威胁建模, STRIDE, 攻击面, 数据流图, 信任边界, threat model

## 描述
使用 STRIDE 方法论进行系统级威胁建模，识别和评估安全威胁。

## 工作流程

### 步骤 1: 系统分解

1. **绘制数据流图 (DFD)**
   - 识别外部实体（用户、第三方服务）
   - 识别进程（应用逻辑、API服务）
   - 识别数据存储（数据库、文件系统、缓存）
   - 识别数据流（API调用、数据传输）

2. **标识信任边界**
   - 互联网 ↔ DMZ
   - DMZ ↔ 内网
   - 应用 ↔ 数据库
   - 用户空间 ↔ 管理员空间

### 步骤 2: STRIDE 分析

对每个 DFD 元素应用 STRIDE 分析：

| 威胁 | 安全属性 | 常见攻击 | 缓解策略 |
|------|---------|---------|---------|
| Spoofing | 认证 | 凭据窃取、会话劫持 | MFA、令牌签名 |
| Tampering | 完整性 | 参数篡改、中间人攻击 | 数字签名、HMAC |
| Repudiation | 不可否认 | 操作否认 | 审计日志、时间戳 |
| Info Disclosure | 机密性 | 数据泄露、侧信道 | 加密、访问控制 |
| DoS | 可用性 | 资源耗尽、放大攻击 | 速率限制、CDN |
| EoP | 授权 | 权限提升、越权 | 最小权限、RBAC |

### 步骤 3: 风险评估矩阵

使用 影响 × 可能性 矩阵：

```
影响\可能性  |  低  |  中  |  高
  高         | 中   | 高   | 紧急
  中         | 低   | 中   | 高
  低         | 信息 | 低   | 中
```

### 步骤 4: 生成威胁模型文档

输出结构化威胁模型，包含：
- 系统架构概述
- DFD 和信任边界
- 每个组件的 STRIDE 分析
- 风险矩阵（排序后）
- 缓解措施和优先级
